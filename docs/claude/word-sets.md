# Правила: Наборы слов

## Структура набора: тема + уровень, 25 слов

| `level` | `name` пример | Доминирующая редкость | Цвет |
|---------|--------------|----------------------|------|
| 1 — Основы | `"Медицина: основы"` | COMMON | серый |
| 2 — Продвинутый | `"Медицина: продвинутый"` | UNCOMMON | зелёный |
| 3 — Углублённый | `"Медицина: углублённый"` | RARE | синий |
| 4 — Профессиональный | `"Медицина: профессиональный"` | EPIC | фиолетовый |
| 5 — Носитель языка | `"Медицина: носитель языка"` | LEGENDARY | жёлтый |

- Разброс внутри набора ≤ 2 смежных уровня редкости
- Несколько наборов одного уровня: `"Медицина: основы"` → `"Медицина: основы 2"` → `"Медицина: основы 3"`
- Дубли между разными темами допустимы; внутри одной темы — запрещены

## Поля WordSetEntity

```kotlin
WordSetEntity(id=1061, languagePair="he-ru", orderIndex=1061,
    name="Медицина: основы",          // "<topic>: <уровень> [N]"
    description="Симптомы, диагнозы", // без CEFR (A1/B1 и т.д.)
    topic="Медицина", level=1)
```

## Поля WordEntity — перевод

- Одно значение: `translation = "врач"`
- Разные значения: `translation = "справедливость / правосудие"` (через ` / `)
- Одно понятие: `translation = "скачивание, загрузка"` (через `, `)
- Максимум 2 варианта; иврит обязательно имеет `transliteration`

## Что блокирует хук (staged файлы)

| ❌ Блокирует | ⚠️ Предупреждает |
|---|---|
| Набор ≠ 25 слов | CEFR в описании |
| Украденные слова внутри темы | Нарушение формулы ID |
| Дублирующийся setId | Дубли имён наборов |
| setId вне блока языка | Транслитерация отсутствует (he-ru) |
| Разброс редкости > 2 уровней | |
| Пустой topic или level=0 | |
| name ≠ topic + уровень | |
| Однокоренные слова в наборе | |

Порядок файлов берётся из `WordRegistry.kt` автоматически.

## Чеклист нового набора

1. `topic` заполнен, `level` 1–5, `name` = `"<topic>: <уровень>"`
2. `description` без CEFR — только тематика
3. Ровно 25 слов; ID = `setId × 100 + позиция`
4. Редкость соответствует уровню, разброс ≤ 2 смежных
5. Нет однокоренных пар; нет дублей внутри темы
6. Иврит: каждое слово имеет `transliteration`
7. Переводы: ` / ` для разных значений, `, ` для синонимов
8. Зарегистрировать в `WordRegistry` → поднять `WordDataVersion`
9. `python find_real_dupes.py` → все проверки зелёные

## Частые ошибки

- Замена дубля «на глаз» → тоже дубль: `grep -rF 'original = "СЛОВО"' data/`
- `grep -w` ненадёжен для иврита — только `grep -F`
- Контекст оборвался → набор < 25 слов: считай в конце
- Exclusive achievement-слова (AchievementCards.ALL_EXCLUSIVE) **можно** добавлять в наборы — они только исключены из PvP-паков
