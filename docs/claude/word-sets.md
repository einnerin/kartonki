# Правила: Наборы слов

> ⚠️ **Область действия.** Все правила в этом документе применяются **только к встроенным наборам** — тем, что зарегистрированы в `WordRegistry.kt` и поставляются с приложением. В будущем планируются пользовательские наборы (UGC, см. [`ugc-plan.md`](ugc-plan.md)) — они живут с флагом `isUserCreated = true`, имеют собственный ID-блок и **не подчиняются** большинству этих правил (формату имени, обязательному `level/topic`, лимиту 25 слов, разбросу редкости, уникальности имён в теме). Валидаторы и pre-commit хуки пропускают пользовательские наборы.

## Структура набора: тема + уровень, 25 слов

**Имя набора = тема.** Всегда. Уровень кодируется только полем `level` (через цвет/прогрессию в UI), а не словом в имени. Различающая информация между сетами одной темы — только в `description`.

| `level` | `name` | `description` пример | Доминирующая редкость | Цвет |
|---------|--------|----------------------|----------------------|------|
| 1 — Основы | `"Медицина"` | `"Тело, боль и самочувствие"` | COMMON | серый |
| 2 — Продвинутый | `"Медицина"` | `"У врача: приём, жалобы, диагноз"` | UNCOMMON | зелёный |
| 3 — Углублённый | `"Медицина"` | `"Диагностика и хирургия"` | RARE | синий |
| 4 — Профессиональный | `"Медицина"` | `"Терминология врача-кардиолога"` | EPIC | фиолетовый |
| 5 — Носитель языка | `"Медицина"` | `"Латынь и редкие диагнозы"` | LEGENDARY | жёлтый |

- Несколько наборов одного уровня: имя одинаковое, `description` у каждого свой и уникальный в пределах темы.
- Порядок наборов в UI: сортировка по `level`. На `id` и `orderIndex` полагаться не стоит — UI группирует по теме и уровню.
- История формата (старый `"<topic>: <уровень> [N]"` отменён 2026-04-24): см. `memory/feedback_set_naming_pattern.md`.

## ⛔ Разброс редкости — строго 2 смежных

**Никогда** не делать «плавную прогрессию» COMMON→LEGENDARY в одном наборе. Только **2 соседних** редкости.

| Level | Допустимые пары |
|-------|----------------|
| 1 | COMMON + UNCOMMON |
| 2 | COMMON + UNCOMMON **или** UNCOMMON + RARE |
| 3 | UNCOMMON + RARE **или** RARE + EPIC |
| 4 | RARE + EPIC **или** EPIC + LEGENDARY |
| 5 | EPIC + LEGENDARY |

❌ `1 COMMON + 5 UNCOMMON + 12 RARE + 5 EPIC + 2 LEGENDARY` — спред 5, нарушение.
✅ `5 UNCOMMON + 20 RARE` — спред 2, ОК.

## Поля WordSetEntity — все обязательные

```kotlin
WordSetEntity(
    id = 1519,
    name = "Медицина",                       // = topic, всегда
    description = "Диагностика и хирургия",  // уникальный в пределах темы, без CEFR (A1/B1 и т.д.)
    languagePair = "he-ru",                  // ОБЯЗАТЕЛЬНО
    topic = "Медицина",                      // ОБЯЗАТЕЛЬНО
    level = 3,                               // 1..5 ОБЯЗАТЕЛЬНО
)
```

**Про `orderIndex`:** поле существует в БД, но **в новых наборах указывать не нужно** — запросы сортируют по `id`, а не по `orderIndex`. В ~739 существующих наборах оно задано равным `id` (избыточно, сохранено из исторических соображений). Для нового набора — просто пропускай, дефолт `0` устроит.

## Поля WordEntity — все обязательные

```kotlin
WordEntity(
    id = 151901,              // формула: setId × 100 + позиция (1..25)
    setId = 1519,
    languagePair = "he-ru",   // совпадает с набором
    rarity = "RARE",          // COMMON | UNCOMMON | RARE | EPIC | LEGENDARY
    original = "אִבְחוּן",
    translation = "диагностика / постановка диагноза",
    transliteration = "ивхун", // ОБЯЗАТЕЛЬНО для he-ru
)
```

### Необязательные runtime-поля для FILL_IN_BLANK

- **`fillInBlankExclusions: List<Long>`** — id слов того же набора, которые подходят в пропуск `example` этого слова (ложные друзья по блоку FILL_IN_BLANK). При создании набора оставлять пустым — заполняется pipeline-ом после фиксации набора. См. [`docs/claude/fill-in-blank-pipeline.md`](fill-in-blank-pipeline.md).
- **`isFillInBlankSafe: Boolean = true`** — ставить `false` только для **form mismatch** (original не совпадает с формой в example — «egg» vs «eggs») или **объективно generic example** («I saw a ___»). Обычную ambiguity (apartment/cottage для house) теперь ловят `fillInBlankExclusions`, помечать такие слова unsafe больше не надо.

- **Translation**: одно значение `"врач"`; разные значения через ` / ` (максимум 2): `"справедливость / правосудие"`; синонимы через `, ` (одно понятие): `"скачивание, загрузка"`.
- **Два слова в наборе с одинаковым translation запрещены** — ученик путается. Если нужны синонимы (cite/refer), уточняй: `cite = "цитировать"`, `refer = "ссылаться / обращаться"`.

## ID-формула (глобальная)

`id = setId × 100 + позиция (1..25)`. Позиция считается **глобально** по всем файлам — если набор раскидан (Immigrant + ImmigrantExtra), нумерация сквозная.

## Политика дублей

- **Внутри набора**: запрещены и `original`, и `translation`.
- **Внутри темы** (разные наборы одной темы, одного языка): `original` не должен повторяться.
- **Между темами** (одно слово в разных темах одного языка): разрешено.

## Однокоренные слова — не в одном наборе

❌ `breakfast` + `bread` в одном наборе — однокоренные.
❌ `cite` + `citation`, `perceive` + `perception`, `electrolysis` + `electrolyte`.
❌ Иврит: слова с общим шорешом (3-буквенным корнем), напр. `שָׁלוֹם` + `שָׁלֵם`.

## Что блокирует валидатор `scripts/validate/find_real_dupes.py`

| ❌ Блокирует | ⚠️ Предупреждает |
|---|---|
| Набор ≠ 25 слов (после удаления дубля) | CEFR в описании |
| Дубль `original` в теме | Нарушение формулы ID |
| Дубль `original`/`translation` в наборе | Дубли пары `(name, description, lang)` |
| Дублирующийся setId | — |
| setId вне блока языка | — |
| Разброс редкости > 2 смежных | — |
| Пустой topic или level=0 | — |
| `name` не начинается с `topic` | — |
| Однокоренные пары в наборе | — |

Порядок файлов берётся из `WordRegistry.kt` автоматически.

## Чеклист перед созданием нового набора

1. **Тема выбрана и обязательно проверена на синонимы среди существующих.** Никаких новых тем без полной сверки списка:
   ```bash
   awk '/languagePair = "<lang>"/{f=1} /topic = "[^"]+"/ && f \
       {match($0, /topic = "[^"]+"/); print substr($0, RSTART+9, RLENGTH-10); f=0}' \
     app/src/main/java/com/example/kartonki/data/WordData*.kt | sort -u
   ```
   Близкие написания (Хай-тек ↔ Хайтек), расширения (Маркетинг ↔ Маркетинг и реклама), синонимы (Здоровье ↔ Здоровье и фитнес) — **не плодить**, использовать существующее. Старая тема выигрывает над «улучшенным» новым именем; переименовать существующее — только осознанной отдельной задачей со слиянием. Защита: `find_real_dupes.py` → `check_topic_synonyms` (warning при близких именах). См. `memory/feedback_no_duplicate_topics.md`.
2. **`name = topic`, без вариаций.** Описание (`description`) — то, чем сет отличается от других в той же теме (подтема, акцент, контекст). Проверь что в той же теме нет сета с таким же `description` — `grep -E 'description = "<твой description>"' app/src/main/java/com/example/kartonki/data/WordData*.kt`.
3. **Все поля WordSetEntity заполнены**: id, name (= topic), description (без CEFR, уникальный в теме), languagePair, topic, level. (orderIndex опционален — не указывай в новых наборах.)
4. **Ровно 25 слов.** Каждое WordEntity имеет id (по формуле), setId, languagePair, rarity, original, translation, + transliteration для he-ru.
5. **Редкость — только 2 смежных уровня** вокруг level. Для L3 это UNCOMMON+RARE **или** RARE+EPIC, не оба.
6. **Нет дублей** в наборе (ни original, ни translation), в теме (проверить другие наборы темы), и однокоренных.
7. **Переводы уникальны** в наборе — если два слова переводятся одинаково, уточни один (`cite → цитировать`, `refer → ссылаться`).
8. **Kotlin-синтаксис**: каждый WordEntity/WordSetEntity заканчивается запятой, нет `,,`, закрывающая скобка listOf не забыта.
9. Зарегистрировать в `WordRegistry.kt` → поднять `WordDataVersion.CURRENT`.
10. `python scripts/validate/find_real_dupes.py` → все проверки зелёные (0 и ❌ и ⚠️).
11. `./gradlew assembleDebug` → `BUILD SUCCESSFUL`.

## Частые ошибки (из реального опыта)

- **Шаблон «1 COM + 5 UNC + 12 RARE + 5 EPIC + 2 LEG»** — плохой, нарушение разброса. 53 моих he-ru наборов были так сделаны и пришлось перерабатывать. Правильно — ровно 2 редкости.
- **Regex `WordEntity\(([^)]+)\)` ломается** на `)` в переводе типа «фабула (хронологический сюжет)». Использовать балансированный парсер (пример — в `fix_spread_all_v2.py`).
- **Дубли при создании нового набора**: перед добавлением слова проверь — нет ли его уже в другом наборе той же темы. `grep -F 'original = "СЛОВО"' app/src/main/java/com/example/kartonki/data/`
- **Контекст оборвался** → меньше 25 слов. Считай в конце.
- **Exclusive achievement-слова** (AchievementCards.ALL_EXCLUSIVE) можно добавлять в наборы — они только исключены из PvP-паков.
