# Правила: Наборы слов (Word Sets / Seed Data)

## ⚠️ Главное: одно слово — один набор в БД

`WordEntity` имеет UNIQUE-индекс на `(original, languagePair)`. Стратегия REPLACE — побеждает **последнее вхождение** в `WordRegistry.allWords`.

Дубль в новом файле «крадёт» слово у старого набора → тот становится 24-словным у всех пользователей.

**Pre-commit хук блокирует коммит при дублях.** Первичная установка: `git config core.hooksPath .githooks`

## Дубли между наборами — ДОПУСТИМЫ (новое правило)

Начиная с переработки he-ru (2026-04), слово может **принадлежать нескольким наборам** через таблицу `word_set_membership`. Это нормально и педагогически полезно.

**Запрещены** дубли только **внутри одной темы** (одинаковый `topic`):
- «Медицина: основы» и «Медицина: продвинутый» — дублей быть не должно
- «Медицина: основы» и «Больница: основы» — дубли допустимы

## Трёхуровневая структура наборов

Каждая тема имеет до 3 уровней. Набор = тема + уровень:

| Уровень | `level` | Редкость слов | Пример имени |
|---------|---------|--------------|--------------|
| Основы | 1 | COMMON + UNCOMMON (A1–B1) | `"Медицина: основы"` |
| Продвинутый | 2 | RARE (B2) | `"Медицина: продвинутый"` |
| Профессиональный | 3 | EPIC + LEGENDARY (C1–C2) | `"Медицина: профессиональный"` |

- Диапазон редкости в наборе — не шире 2 смежных уровней (как раньше)
- Если слов много — «Медицина: основы 1» и «Медицина: основы 2»
- `topic` в `WordSetEntity` = тема без уровня, например `"Медицина"`
- Используется для группировки в UI

## Поля WordSetEntity

```kotlin
WordSetEntity(
    id          = 1061,
    languagePair = "he-ru",
    orderIndex  = 1061,
    name        = "Медицина: основы",   // отображаемое имя
    description = "Базовая медлексика A1/B1",
    topic       = "Медицина",           // для группировки
    level       = 1,                    // 1/2/3, 0 = не задан
)
```

## Процесс при срабатывании хука

Хук покажет: `'слово' stolen by set NEW from set OLD`. Дальше:
1. Прочитай оба набора: `grep -A3 "setId = X," WordDataXxx.kt`
2. Реши где уместнее: тема → уровень CEFR → практичность слова
3. **Слово остаётся в OLD** → замени в NEW на свободное слово
4. **Слово переходит в NEW** → найди замену для OLD того же уровня CEFR
5. Проверь: `python find_real_dupes.py` → должно быть `0 stolen words`

Найти свободные слова: `python list_set_words.py` или `grep -rF 'original = "СЛОВО"' data/`

## Чеклист перед добавлением набора

1. `name`/`description`/`topic` — только на русском
2. 25 слов ровно: `grep -c "setId = XXXX," WordDataXxx.kt`
3. ID слова = `setId × 100 + position`; setId не пересекается с `languageIdBlocks`
4. `topic` заполнен (например `"Медицина"`) и `level` 1/2/3
5. Диапазон редкости — не шире 2 смежных уровней
6. Нет слов из `AchievementCards.ALL_EXCLUSIVE`
7. Нет дублей внутри той же темы (`topic`)
8. Зарегистрировать в `WordRegistry.allSets` и `WordRegistry.allWords`
9. Поднять `WordDataVersion.CURRENT` на 1

## Редкость

COMMON=A1 · UNCOMMON=A2-B1 · RARE=B2 · EPIC=C1 · LEGENDARY=C2+
Объективный CEFR-уровень, не интуиция. Проверять: `grep -A3 'original = "СЛОВО"' data/ | grep rarity`

## Что проверяет хук (pre-commit)

`find_real_dupes.py` запускается автоматически при `git commit` если изменились WordData*.kt.

| Проверка | Поведение |
|---|---|
| Украденные слова **внутри одной темы** | ❌ блокирует |
| Набор ≠ 25 слов | ❌ блокирует |
| Дублирующиеся `setId` в разных файлах | ❌ блокирует |
| `setId` вне языкового блока | ❌ блокирует |
| Разброс редкости в staged файлах (> 2 смежных) | ❌ блокирует |
| Пустой `topic` или `level = 0` в staged файлах | ❌ блокирует |
| Однокоренные слова в одном staged наборе | ❌ блокирует |
| Украденные слова **между разными темами** | ℹ️ допустимо |
| CEFR в описании набора (A1/B1 и т.д.) | ⚠️ предупреждение |
| Формула ID, дубли имён | ⚠️ предупреждение |

**Порядок файлов берётся из `WordRegistry.kt` автоматически.** Добавил файл в реестр — хук сразу его видит.

## Аудит

```bash
python find_real_dupes.py   # главная проверка — запускать перед коммитом

```

## Частые ошибки

- Замена дубля выбрана «на глаз» → замена тоже дубль — всегда grep перед записью
- `grep -w` ненадёжен для иврита с огласовками — только `grep -F`
- Написал < 25 слов из-за того что контекстное окно кончилось — считать в конце
- Забыл заполнить `topic` или `level` — без них UI не сгруппирует набор
- Кросс-тематические дубли (врач в Медицине и Больнице) — **допустимы**, хук не блокирует
- В описании набора **не писать A1/B1/C1** — уровень передаётся цветом карточки (COMMON/RARE/EPIC)
- Пример описания: `"Договоры, судопроизводство, юридические термины"` — без CEFR
- Однокоренные слова в одном наборе — **запрещены**, хук поймает
