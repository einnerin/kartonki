# Правила: Наборы слов (Word Sets / Seed Data)

## Структура набора

Набор = одна тема + один уровень сложности. Три уровня:

| `level` | Название | Редкость слов | Пример `name` |
|---------|----------|--------------|---------------|
| 1 | Основы | COMMON, UNCOMMON | `"Медицина: основы"` |
| 2 | Продвинутый | RARE | `"Медицина: продвинутый"` |
| 3 | Профессиональный | EPIC, LEGENDARY | `"Медицина: профессиональный"` |

- Разброс редкости внутри набора — не шире **2 смежных уровней** (COMMON+UNCOMMON ✓, RARE+EPIC ✗)
- Слов в наборе: ровно **25**
- Если на один уровень темы слов много: `"Медицина: основы"`, `"Медицина: основы 2"`, `"Медицина: основы 3"`

## Поля WordSetEntity

```kotlin
WordSetEntity(
    id           = 1061,
    languagePair = "he-ru",
    orderIndex   = 1061,
    name         = "Медицина: основы",       // <topic>: <уровень> [N]
    description  = "Симптомы, диагнозы, лечение",  // БЕЗ A1/B1/C1 — только тематика
    topic        = "Медицина",               // тема без уровня, для группировки в UI
    level        = 1,                        // 1 / 2 / 3
)
```

**Правила имени:**
- Всегда `"<topic>: <уровень>"` — пробел после двоеточия обязателен
- `topic` и часть после двоеточия должны совпадать с полями `topic` и `level`
- При нескольких наборах одного уровня: первый без номера, далее ` 2`, ` 3`

**Правила описания:**
- Только тематика, без CEFR-нотации (A1/B1 и т.д.) — уровень уже передаётся цветом карточки
- ✅ `"Симптомы, диагнозы, лечение, медицинские процедуры"`
- ❌ `"Базовая медлексика A1/B1"`

## Правила перевода (поле `translation`)

Режим обучения — выбор из вариантов (не текстовый ввод), поэтому `translation` — строка для отображения.

**Если у слова одно основное значение:**
```kotlin
translation = "справедливость"
```

**Если два значения реально используются в разных контекстах** — через ` / `:
```kotlin
translation = "справедливость / правосудие"   // justice
translation = "разработка / развитие"          // development
translation = "право / правый"                 // right
```

**Если одно понятие, несколько близких слов** — через `, `:
```kotlin
translation = "экран, монитор"
translation = "скачивание, загрузка"
```

**Когда НЕ добавлять альтернативный перевод:**
- Значения почти одинаковы — выбери одно, точнее подходящее по контексту набора
- Больше двух вариантов — выбери самый частотный
- Редкое, устаревшее или специализированное значение

> Если в будущем появится текстовый ввод — добавим отдельное поле `alternativeTranslations`.
> До тех пор хранится в одной строке.

## Дубли между наборами

- **Запрещены** внутри одной темы (`topic`): «Медицина: основы» и «Медицина: продвинутый» — дублей нет
- **Допустимы** между разными темами: «Медицина: основы» и «Больница: основы» — одно слово в обоих нормально

Слово хранится в БД один раз (`UNIQUE` по `original + languagePair`). Принадлежность к нескольким наборам — через таблицу `word_set_membership`.

## Поля WordEntity

```kotlin
WordEntity(
    id            = 106101,               // setId × 100 + позиция (1–25)
    setId         = 1061,                 // «родной» набор
    languagePair  = "he-ru",
    original      = "רוֹפֵא",
    transliteration = "rofe",            // только для иврита и нелатинских алфавитов
    translation   = "врач / доктор",     // основной перевод; альтернативы через / или ,
    definition    = "...",               // определение на языке оригинала (необязательно)
    definitionNative = "...",            // определение на русском (необязательно)
    example       = "...",               // пример предложения (необязательно)
    exampleNative = "...",
    rarity        = "COMMON",            // COMMON / UNCOMMON / RARE / EPIC / LEGENDARY
    pos           = "noun",              // часть речи (необязательно)
)
```

## Что проверяет хук автоматически

`find_real_dupes.py` запускается при `git commit` если изменились `WordData*.kt`.
Порядок файлов берётся из `WordRegistry.kt` — не нужно обновлять вручную.

| Проверка | Поведение |
|---|---|
| Украденные слова внутри одной темы | ❌ блокирует |
| Набор ≠ 25 слов | ❌ блокирует |
| Дублирующиеся `setId` в разных файлах | ❌ блокирует |
| `setId` вне языкового блока (en-ru 1–999, he-ru 1001–1999) | ❌ блокирует |
| Разброс редкости > 2 смежных уровней (staged) | ❌ блокирует |
| Пустой `topic` или `level = 0` (staged) | ❌ блокирует |
| Имя набора не соответствует `topic` / `level` (staged) | ❌ блокирует |
| Однокоренные слова в одном наборе (staged) | ❌ блокирует |
| Украденные слова между разными темами | ℹ️ допустимо |
| CEFR в описании набора (staged) | ⚠️ предупреждение |
| Нарушение формулы ID, дубли имён (staged) | ⚠️ предупреждение |

## Чеклист перед добавлением набора

1. `name` = `"<topic>: <уровень>"`, `topic` заполнен, `level` = 1/2/3
2. `description` — тематика без CEFR (без A1/B1/C1 и т.д.)
3. Ровно 25 слов
4. ID слова = `setId × 100 + позиция`; `setId` в диапазоне языка
5. Редкость — не шире 2 смежных уровней (см. таблицу выше)
6. Нет однокоренных пар внутри набора
7. Нет дублей внутри той же темы (`topic`)
8. Нет слов из `AchievementCards.ALL_EXCLUSIVE`
9. Переводы: один главный; альтернативы через ` / ` (разные значения) или `, ` (одно понятие)
10. Зарегистрировать в `WordRegistry.allSets` и `WordRegistry.allWords`
11. Поднять `WordDataVersion.CURRENT` на 1

## Редкость слов

| Метка | CEFR | Описание |
|-------|------|----------|
| COMMON | A1 | Базовые слова, знают дети |
| UNCOMMON | A2–B1 | Разговорный уровень |
| RARE | B2 | Продвинутый уровень |
| EPIC | C1 | Академический / профессиональный |
| LEGENDARY | C2+ | Редкие, архаичные, узкоспециальные |

Редкость — **объективный CEFR**, не интуиция. Менять только если метка объективно неверна.

## Частые ошибки

- Замена дубля выбрана «на глаз» → сама оказалась дублем — всегда проверяй: `grep -rF 'original = "СЛОВО"' data/`
- `grep -w` ненадёжен для иврита с огласовками — только `grep -F`
- Контекстное окно кончилось на середине набора → написано < 25 слов — считай в конце
- Написал `description = "... B2/C1"` — хук предупредит, убери CEFR
- Кросс-тематические дубли (врач в Медицине и Больнице) — допустимы, хук не блокирует
- В альтернативном переводе > 2 вариантов — это сигнал что слово слишком широкое для набора
