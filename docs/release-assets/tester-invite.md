# Приглашение тестерам — Internal testing

## Как работает Internal testing

1. Ты в Play Console добавляешь email тестера в список Internal testing track (до 100 адресов)
2. Этот email **должен быть Google-аккаунтом** (Gmail или любой email привязанный к Google)
3. После публикации релиза Console генерирует **opt-in URL** вида `https://play.google.com/apps/internaltest/4699999999999999999`
4. Тестер открывает этот URL **на том устройстве где залогинен под этим Google-аккаунтом**, нажимает «Become a tester»
5. После этого приложение появляется в Play Store по поиску `com.einerin.kartonki` или по прямой ссылке
6. Установка идёт **через стандартный Play Store** (не sideload), все обновления автоматические
7. На карточке будет пометка «Closed test» / «Internal test»

**Важно:** opt-in нужно делать только один раз на каждый аккаунт. После этого аккаунт постоянно видит internal-сборку.

## Шаблон сообщения тестеру (Telegram / WhatsApp / email)

### Короткий вариант (мессенджер)

```
Привет! Я делаю приложение Kartonki — учить английский и иврит через карточки и квизы. Запускаю тестирование, нужны 3-4 первых пользователя.

Что нужно:
1. Дай Gmail-адрес, который у тебя на телефоне
2. Я добавлю тебя в список тестеров
3. Открой ссылку которую я пришлю → «Become a tester» → установи через Play Store
4. Поиграй 5-10 минут, скажи что понравилось и что сломалось

Бесплатно, без рекламы, никакие данные никуда не идут. Если не зайдёт — напиши прямо, починю или забью.
```

### Полный вариант (email)

```
Тема: Тестирование Kartonki — нужен твой фидбэк

Привет, [имя],

Запускаю Internal testing своего Android-приложения Kartonki — это игровой тренажёр для изучения английских и ивритских слов, с карточками с редкостями, квизами и PvP с друзьями.

Ищу 3-4 первых тестеров чтобы поймать очевидные баги до публичного релиза. Если интересно поучаствовать — нужно от тебя:

1. **Gmail-адрес** который у тебя залогинен на Android-телефоне (любой Google-аккаунт подойдёт)
2. После того как добавлю тебя в список — пришлю **opt-in URL**, открой его на том телефоне, нажми «Become a tester»
3. Через 1-15 минут приложение появится в Play Store, ставь как обычное
4. Поиграй 10-20 минут: попробуй разные темы, квизы, может быть PvP с другим тестером

Что важно знать:

- Не запрашивает сомнительных разрешений (только инет)
- Анонимно играть тоже можно, Google-логин нужен только для онлайн-PvP
- Политика конфиденциальности: https://einnerin.github.io/kartonki/legal/privacy-policy
- Если что-то падает — дай знать, починю в следующей сборке

Готов? Скинь Gmail в ответ.

Спасибо что согласился попробовать!
[твоё имя]
einerin40@gmail.com
```

## Что просить тестировать (фокус-лист)

Чтобы фидбэк был полезным а не «ну норм», направляй тестеров:

| Сценарий | Что проверить |
|---|---|
| Первый запуск | Splash, login, выбор языковой пары — всё ли понятно с порога |
| Изучение нового сета | Открыть тему, пройти 10-20 карточек, понять алгоритм повторений |
| Квизы | Проверить что варианты ответов адекватны (нет неоднозначных), нет опечаток |
| PvP local | Передать телефон друг другу, играть на одном устройстве |
| PvP online | Найти соперника через matchmaking — реально ли матч проходит до конца |
| Коллекция | Открыть пак, посмотреть как карточка с редкостью выглядит |
| Достижения | Какие открылись после первой сессии — мотивирует или нет |
| Звук / TTS | Произношение слов работает? Не лагает? |
| Тёмная/светлая тема | Если устройство переключается — UI не ломается? |

## Сбор фидбэка

Самое лёгкое — попросить кидать в Telegram/WhatsApp скриншоты + 1-2 фразы. Если хочешь систематичности:

- Создать Google Form с полями: «что понравилось / что бесит / что сломалось / оценка от 1 до 5»
- Или Telegram-канал «Kartonki feedback» куда тестеры скидывают всё подряд

Для 3-4 человек на Internal — Form избыточен. Просто личка.

---

## Что дальше после Internal

Через 1-2 недели Internal:
1. Собрал критические баги — пофикси, выпусти v0.1.1 в Internal
2. Когда стабильно — дублируй релиз в **Open testing** (Early access, публично)
3. Параллельно начни **Closed testing** с 12 тестерами — это запускает 14-дневный таймер на Production

Подробности в `docs/claude/release-plan.md`.

## Полезные команды

Снять отзыв с устройства тестера (через USB):
```bash
adb logcat -d | grep -i 'kartonki\|FATAL\|AndroidRuntime' > tester_log.txt
```

Удалить debug-сборку с устройства тестера если у него стоит и debug, и release одновременно (конфликт по applicationId):
```bash
adb uninstall com.einerin.kartonki
# потом тестер ставит свежую через Play Store
```
