# Скриншоты для Play Console — инструкция

Play Console для phone требует **минимум 2 скриншота**, рекомендует 4-8. Хороший набор показывает **разнообразие механик**: чтобы пользователь за 3 секунды на карточке приложения понял «это словарь + игра + соревнование с друзьями».

---

## Технические требования Play Store

| Параметр | Значение |
|---|---|
| Формат | JPEG или 24-bit PNG (без альфа-канала) |
| Размер короткой стороны | от 320 до 3840 px |
| Размер длинной стороны | максимум **в 2 раза** больше короткой (т.е. 16:9 ≤ aspect ≤ 9:16) |
| Минимальное число | 2 |
| Максимальное число | 8 |
| Рекомендуемое | 4-8 |

**Целевое разрешение:** 1080×2400 portrait (стандартный современный смартфон). Можно и 1080×1920. Главное — все скриншоты в **одинаковом** разрешении.

Tablet-скриншоты (7" и 10") **не обязательны** на этапе Internal testing. Когда дойдёшь до Production — желательно добавить, но не блокер.

---

## Какие экраны снимать (порядок = порядок в Play Store)

Порядок в листинге важен: первые 2 скрина видны на карточке поиска без свайпа.

| # | Экран | Что показать | Зачем |
|---|---|---|---|
| **1** | **Главный/Study session с карточкой слова** | Карточка с readable словом, переводом, редкостью видна (золотая рамка для RARE+) | Сразу понятно: это про изучение слов, и это карточная игра |
| **2** | **Квиз FILL_IN_BLANK** | Предложение с пропуском + 4 варианта ответа, один уже выбран (или активный hover) | Показывает интерактив, что это не просто flip-карточки |
| **3** | **Список тем / Set picker** | Скролл с 5-7 тематиками, видны названия, прогресс-бары | Раскрывает объём: десятки тем, есть прогресс |
| **4** | **PvP local или online** | Экран матча: твоя карточка, ходы соперника, счёт | Differentiator над Duolingo/Anki — игра с друзьями |
| **5** | **Коллекция / открытие пака** | Анимация флипа карты с rarity-эффектом, или коллекция с разными редкостями | ККИ-аспект, эстетика |
| **6** | **Достижения / streak** | Список ачивок, видна серия дней | Мотивация, retention loop |

Если соберёшь все 6 — отлично. Если устанешь после 4 — тоже норм для Internal track.

**Опционально (если будет время):**
- 7. Settings с переключателем языковой пары (en-ru ↔ he-ru) — чтобы видно было что есть оба языка
- 8. Splash screen или Login — для эстетики

## Как снимать

### Вариант A — эмулятор (рекомендуется для idem-результата)

В Android Studio:
1. Tools → Device Manager → запусти эмулятор **Pixel 7 Pro API 34** (1440×3120 — после ресайза до 1080 даст идеально чистый кадр)
2. Установи свежий debug-build через Run
3. На эмуляторе — открой нужный экран
4. **Extended Controls** (три точки справа от toolbar эмулятора) → **Camera/Screenshot icon** → файл попадёт в `~/Desktop` или путь, который указан в Settings → Screenshots
5. Альтернатива: **⌘ + S** в фокусе эмулятора (macOS) или просто кнопка Camera в toolbar эмулятора

Преимущество: status bar чистый (нет реальных уведомлений), время фиксируется, заряд 100%.

### Вариант B — телефон

```bash
# подключи телефон по USB, USB-debugging on
adb shell screencap -p /sdcard/screen.png
adb pull /sdcard/screen.png ./screenshots/01-card.png
adb shell rm /sdcard/screen.png
```

**На Windows + Git Bash** этот путь надёжнее чем `adb exec-out screencap -p > screen.png`, потому что Bash может побить бинарный stream символами CRLF. Через `screencap → pull` PNG идёт через файл, без преобразований.

Для удобства — готовый скрипт `scripts/release/shot.sh`:

```bash
# по одному кадру за вызов — кладёт в docs/release-assets/screenshots/<name>.png
bash scripts/release/shot.sh 01-card
bash scripts/release/shot.sh 02-quiz
bash scripts/release/shot.sh 03-themes

# чистый статус-бар: 12:00, battery 100%, wifi 4, без уведомлений
bash scripts/release/shot.sh demo-on
# вернуть статус-бар в нормальный режим
bash scripts/release/shot.sh demo-off
```

### Вариант C — release-сборка на телефоне

Сейчас на телефоне **release-сборка** от 2026-04-29 (без in-app deletion флоу). Если хочешь снимать с актуальной версии (включая кнопку «Удалить аккаунт» в Settings — хотя она не идёт в скрины листинга), накати свежую:

```bash
bash scripts/release/install-release.sh --build
# или, если AAB/APK уже свежий, просто:
bash scripts/release/install-release.sh
```

Цвета и иконка release-сборки идентичны тому что увидят пользователи. Проверь только что app в нормальном состоянии (есть прогресс, есть карточки в коллекции, серия ≥ 2 дня).

Если хочешь снимать с **debug** — `adb uninstall com.einerin.kartonki` и поставить debug заново через Run в Android Studio.

## Подготовка приложения перед снятием

Чтобы скриншоты выглядели «обжитыми» а не пустыми:

1. **Зайди в аккаунт** (Google Sign-In) — иначе на главном экране будет призыв логиниться, а нам нужны рабочие экраны
2. **Поучи 2-3 темы**, чтобы:
   - На главной была серия (streak ≥ 2 дня)
   - В коллекции были разные редкости
   - В прогрессе видны % выученного
3. **Создай 1-2 колоды** для PvP — чтобы был экран «выбор колоды»
4. **Сыграй PvP local** хотя бы один раз — для скрина, где видно реальный матч (не лендинг)
5. **Открой 2-3 пака** — для красивой коллекции
6. **Разблокируй несколько ачивок** простыми действиями

## Чистка status bar

Стандартный фокус — статус-бар не должен отвлекать.

**На эмуляторе** — он по умолчанию чистый.

**На телефоне:**
- Включи Do Not Disturb (выключит входящие уведомления)
- Подключи к зарядке (значок full battery)
- Включи Wi-Fi (хотя бы значок)
- Отключи signal indicator если возможно (через demo mode):
  ```bash
  # включить demo mode (только debug-сборки Android, может не работать на retail)
  adb shell settings put global sysui_demo_allowed 1
  adb shell am broadcast -a com.android.systemui.demo -e command enter
  adb shell am broadcast -a com.android.systemui.demo -e command battery -e level 100 -e plugged false
  adb shell am broadcast -a com.android.systemui.demo -e command network -e wifi show -e level 4
  adb shell am broadcast -a com.android.systemui.demo -e command notifications -e visible false
  adb shell am broadcast -a com.android.systemui.demo -e command clock -e hhmm 1200
  ```
  Если эта команда не сработала на твоём девайсе — забей, обычный status bar тоже OK.

## Куда складывать

Финальные скриншоты → `docs/release-assets/screenshots/01-card.png`, `02-quiz.png`, ...

Этот путь **в репо** (под git). Сами скрины весят ~500KB-2MB каждый, общий объём 6-8 MB — нормально для git.

Если получится несколько вариантов одного экрана — клади в подпапку `screenshots/drafts/`, в финальной директории оставь только то что пойдёт в Console.

## Резюме

1. Подготовь app (см. выше)
2. Запусти эмулятор Pixel 7 Pro или возьми телефон
3. Сними 4-6 кадров по списку выше
4. Складывай в `docs/release-assets/screenshots/NN-name.png`
5. Когда готово — пингуй меня, проверим что разрешения одинаковые и нет приватных данных в кадре, и пойдём в Console грузить
